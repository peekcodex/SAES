The Four AES Transformations

Each round of AES applies four mathematical operations to the 16-byte state matrix, providing both confusion and diffusion.

01
⬡
Substitute Bytes

Every byte is replaced via a non-linear S-box lookup table, derived from the multiplicative inverse in GF(2⁸). This creates maximum confusion.

Confusion
02
⇄
Shift Rows

Row r of the matrix is cyclically shifted left by r positions. This ensures bytes from different columns are spread across the state.

Transposition
03
⊗
Mix Columns

Each column is multiplied by a fixed matrix in GF(2⁸) finite field arithmetic, guaranteeing massive diffusion — the avalanche effect.

Diffusion

Add Round Key

The state matrix is XOR-ed with the 128-bit round key derived from your secret password, tying security directly to the key schedule.

The 16-byte plaintext is arranged into a 4×4 (or here, 2×2 for illustration) state matrix. Each cell holds one byte in hexadecimal.

Our example starts with the state [19, 3d, a4, 7e] alongside the round key [2b, 28, ab, 09] that will be applied at the end of this round.

SubBytes — S-box Substitution

Each byte is independently looked up in the AES S-box — a precomputed 256-entry table built from the multiplicative inverse in GF(2⁸) followed by an affine transformation.

This step provides Confusion: there is no linear relationship between input and output bytes, making the cipher resistant to algebraic attacks.

// AES S-box lookups
19 → d4  (row 1, col 9)
3d → 27  (row 3, col d)
a4 → 49  (row a, col 4)
7e → f3  (row 7, col e)

An inverse S-box exists for decryption, designed so the transformation is its own inverse when applied twice.

ShiftRows — Cyclic Row Shift

Each row r is cyclically shifted left by r bytes. In a full 4×4 AES matrix: row 0 stays, row 1 shifts by 1, row 2 by 2, row 3 by 3.

For our 2×2 example, row 1 shifts left by 1 position. This ensures each column of the output state contains bytes from every column of the input — providing inter-column diffusion.

// 2×2 ShiftRows rule
Row 0: [ d4, 49 ] → [ d4, 49 ]  (no shift)
Row 1: [ 27, f3 ] → [ f3, 27 ]  (shift ←1)

// New state after ShiftRows
[ d4, 49 ]
[ f3, 27 ]

Note how f3 (originally column 1) now appears in column 0, mixing the columns for the subsequent MixColumns step.

AddRoundKey — XOR with Key

The state is XOR-ed bit-by-bit with the 128-bit round key. This is the only step that directly incorporates the secret key, so security depends entirely on key secrecy.

XOR is its own inverse: applying the same key again recovers the original. This means encryption and decryption share the same operation.

// State ⊕ Round Key
bd ⊕ 2b = 96
95 ⊕ 28 = bd
27 ⊕ ab = 8c
6e ⊕ 09 = 67

// Final state after this round
[ 96 8c ]
[ bd 67 ]
